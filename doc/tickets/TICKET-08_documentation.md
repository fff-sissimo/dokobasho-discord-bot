# TICKET-08: READMEと最終ドキュメント整備

## 概要

他の開発者や運用者がプロジェクトを容易に理解し、デプロイ・運用できるように、`README.md`に必要な情報を追記する。

## ToDo

- [ ] **デプロイ手順**:
    - [ ] `git clone` から `npm install` までの手順を記述する。
    - [ ] `.env` ファイルのセットアップ方法を記述する。
    - [ ] スラッシュコマンドの登録方法 (`npm run deploy-commands`) を記述する。
    - [ ] ボットとスケジューラの起動方法 (`npm start`, `npm run start-scheduler`) を記述する。
    - [ ] Dockerを使ったデプロイ手順もあれば追記する。
- [ ] **運用上の注意**:
    - [ ] タイムゾーン設定の重要性について言及する。
    - [ ] Google Service Account のキー管理に関する注意を記述する。
    - [ ] `status=failed` になったリマインダーの手動再送や確認の方法を記述する。
    - [ ] ログの確認方法について記述する。

## 実装のヒント

- プロジェクトを全く知らない人が、READMEを読むだけでセットアップから起動まで完結できることを目指す。
